# ПОЛНЫЙ И ДЕТАЛЬНЫЙ АНАЛИЗ ПРИЛОЖЕНИЯ.
## ИЗБРАННОЕ — принципы добавления/удаления трека
Основные принципы добавления и удаления трека в ИЗБРАННОЕ:
Лайк/анлайк (⭐) = избранность трека.
Состояние ⭐ синхронизируется мгновенно во всех местах (родной альбом, «Избранное», другие списки).

Удаление трека из окна «Избранное»
Это отдельное действие – реализовано как двухступенчатая защита от случайного удаления.

Подробная логика работы
⭐ в родном альбоме (источник истины):
- Поставили ⭐ — трек появляется в «Избранном» как активный.
- Сняли ⭐ — трек полностью исчезает из «Избранного» (даже если был неактивным, «без следа»).

⭐ в окне «Избранное»:
- Сняли ⭐ с активной строки — она становится серой (inactive), ⭐ снимается и в «Избранном», и в родном альбоме.
- Снятие ⭐ НЕ удаляет строку окончательно, а только переводит её в неактивное состояние (“буфер”).

Поведение неактивной (серой) строки в «Избранном»:
- Клик по серой строке (не по звезде) — открывает модалку с вариантами:
  - Вернуть в избранное — строка становится активной, ⭐ сразу восстанавливается везде.
  - Удалить — строка исчезает из «Избранного» окончательно. В родном альбоме ⭐ при этом снята.
- Клик по ⭐ на серой строке — быстрое восстановление без модалки, строка мгновенно становится активной с ⭐.
- Удаление строки из «Избранного» возможно только через модалку (или “без следа” через анлайк в родном альбоме).

Синхронизация:
- Любое изменение ⭐ приводит к мгновенному обновлению состояния во всех представлениях без перезагрузок.
- В окне «Избранное»: снятие ⭐ на любой строке приводит к неактивности этой строки и синхронно снимает ⭐ в родном альбоме.
- Полное удаление строки возможно только через модалку.

Перестройка плейлиста «Избранное»:
- Любые переходы трека в inactive, восстановление или удаление мгновенно отражаются на составе плейлиста.
- Проигрывание текущей песни НЕ обрывается из-за перестройки плейлиста, изменения происходят “в фоне”.

Особое правило для текущего трека:
- Если трек становится inactive в «Избранном» во время воспроизведения — сразу переключается на следующий активный трек.
- Если это был единственный активный трек — плеер переходит в режим стоп, музыка останавливается.

Родной альбом всегда решает окончательно:
- Если снять ⭐ с трека в родном альбоме — он полностью исчезает из «Избранного», даже если находился там как неактивная (серая) строка.

Восстановление из неактивного:
- При установке ⭐ на треке в родном альбоме, если соответствующая неактивная строка в «Избранном» уже есть — она становится активной.

Итоги
- Нет возможности случайного удаления — шаг удаления всегда подтверждается модалкой.
- Все изменения состояния ⭐ мгновенно синхронизированы во всех точках.
- Логика отображения и поведения не содержит противоречий: одно и то же действие всегда приводит к одному ожидаемому результату.
- Родной альбом является финальным арбитром присутствия трека в «Избранном».

Назначение кнопки F (Только избранные)
Кнопка F включает/выключает режим Favorites Only для текущего плейлиста воспроизведения (playing-плейлиста).
Она не ставит/не снимает ⭐ у треков, а только ограничивает, по каким трекам можно ходить через Next / Prev / автопереход и какие треки участвуют в Shuffle.
Единый принцип (полный плеер и мини-плеер)
Полный и мини-плеер управляют одним и тем же playing-плейлистом.
Поведение кнопки F полностью одинаковое в обоих режимах: всегда применяется к playing, а не к открытому на экране списку.
Что считается “доступным треком”
Доступные треки — это те, которые участвуют в:
Next / Prev
автопереходе по окончанию трека
Shuffle-порядке (если shuffle включён)
Правило доступности:
F = OFF: доступен обычный набор playing-плейлиста, но с исключениями контекста (см. “ИЗБРАННОЕ” ниже).
F = ON: доступны только избранные (⭐) треки playing-плейлиста.
Контекст: playing = обычный альбом
Избранность определяется по ⭐ в этом альбоме (в хранилище лайков).
Состояние: Shuffle OFF → Нажатие F
OFF → ON
Если в альбоме нет ни одного ⭐ трека: показать уведомление «отметьте ⭐ понравившийся трек», кнопка остаётся OFF, ничего больше не меняется.
Если ⭐ есть: кнопка становится ON, доступный набор сужается до ⭐, Next/Prev/автопереход работают только по ⭐.
ON → OFF
Кнопка становится OFF, доступны все треки альбома, Next/Prev/автопереход работают по всем трекам.
Состояние: Shuffle ON → Нажатие F
OFF → ON
Если ⭐ нет: уведомление «отметьте ⭐ понравившийся трек», кнопка остаётся OFF.
Если ⭐ есть: кнопка ON, доступный набор = ⭐, shuffle-порядок пересобирается только по ⭐, Next/Prev/автопереход идут по этому shuffle-порядку.
ON → OFF
Кнопка OFF, доступный набор = все треки, shuffle-порядок пересобирается по полному набору, Next/Prev/автопереход идут по нему.
Контекст: playing = ИЗБРАННОЕ (плейлист избранного)
В “ИЗБРАННОЕ” есть элементы active и inactive:
active: трек реально в избранном (⭐ есть) — участвует в воспроизведении
inactive: ⭐ снята, строка существует только для быстрого возврата/удаления через модалку — никогда не участвует в воспроизведении
Базовое правило (важно): inactive всегда пропускаются в Next/Prev/автопереходе и никогда не попадают в доступные наборы и shuffle-порядок — при любом состоянии F.
Состояние: Shuffle OFF → Нажатие F
OFF → ON
Если нет active (в избранном нечего играть): показать уведомление «отметьте ⭐ понравившийся трек», кнопка остаётся OFF.
Если active есть: кнопка ON, доступный набор = active (по факту он и так такой), Next/Prev/автопереход ходят только по active.
ON → OFF
Кнопка OFF, доступный набор остаётся = active (inactive всё равно исключены), Next/Prev/автопереход продолжают ходить только по active.
Состояние: Shuffle ON → Нажатие F
OFF → ON
Если нет active: уведомление «отметьте ⭐ понравившийся трек», кнопка остаётся OFF.
Если active есть: кнопка ON, shuffle-порядок пересобирается по active, Next/Prev/автопереход идут по нему.
ON → OFF
Кнопка OFF, shuffle-порядок пересобирается по active (потому что inactive всё равно исключены), Next/Prev/автопереход идут по нему.
Инварианты (обязательные требования)
Нажатие F никогда не меняет состав избранного (⭐ не ставится и не снимается).
Если нет доступных избранных для включения режима (нет ⭐ в альбоме / нет active в избранном плейлисте): показывается уведомление «отметьте ⭐ понравившийся трек», и F не включается (остаётся OFF).
inactive в ИЗБРАННОМ — это UI-элемент для возврата/удаления, он не должен попадать ни в какие плейлисты воспроизведения, ни в Next/Prev, ни в Shuffle.
