# ПОЛНЫЙ И ДЕТАЛЬНЫЙ АНАЛИЗ ПРИЛОЖЕНИЯ.
## ИЗБРАННОЕ — принципы добавления/удаления трека
Основные принципы добавления и удаления трека в ИЗБРАННОЕ:
Лайк/анлайк (⭐) = избранность трека. Состояние ⭐ синхронизируется мгновенно во всех местах (родной альбом, «Избранное», другие списки).

Удаление трека из окна «Избранное». Это отдельное действие – реализовано как двухступенчатая защита от случайного удаления. Подробная логика работы: 
⭐ в родном альбоме (источник истины):
- Поставили ⭐ — трек появляется в «Избранном» как активный.
- Сняли ⭐ — трек полностью исчезает из «Избранного» (даже если был неактивным, «без следа»).

⭐ в окне «Избранное»:
- Сняли ⭐ с активной строки — она становится серой (inactive), ⭐ снимается и в «Избранном», и в родном альбоме.
- Снятие ⭐ НЕ удаляет строку окончательно, а только переводит её в неактивное состояние (“буфер”).

Поведение неактивной (серой) строки в «Избранном»:
- Клик по серой строке (не по звезде) — открывает модалку с вариантами:
  - Вернуть в избранное — строка становится активной, ⭐ сразу восстанавливается везде.
  - Удалить — строка исчезает из «Избранного» окончательно. В родном альбоме ⭐ при этом снята.
- Клик по ⭐ на серой строке — быстрое восстановление без модалки, строка мгновенно становится активной с ⭐.
- Удаление строки из «Избранного» возможно только через модалку (или “без следа” через анлайк в родном альбоме).

Синхронизация:
- Любое изменение ⭐ приводит к мгновенному обновлению состояния во всех представлениях без перезагрузок.
- В окне «Избранное»: снятие ⭐ на любой строке приводит к неактивности этой строки и синхронно снимает ⭐ в родном альбоме.
- Полное удаление строки возможно только через модалку.

Перестройка плейлиста «Избранное»:
- Любые переходы трека в inactive, восстановление или удаление мгновенно отражаются на составе плейлиста.
- Проигрывание текущей песни НЕ обрывается из-за перестройки плейлиста, изменения происходят “в фоне”.

Особое правило для текущего трека:
- Если трек становится inactive в «Избранном» во время воспроизведения — сразу переключается на следующий активный трек.
- Если это был единственный активный трек — плеер переходит в режим стоп, музыка останавливается.

Родной альбом всегда решает окончательно:
- Если снять ⭐ с трека в родном альбоме — он полностью исчезает из «Избранного», даже если находился там как неактивная (серая) строка.

Восстановление из неактивного:
- При установке ⭐ на треке в родном альбоме, если соответствующая неактивная строка в «Избранном» уже есть — она становится активной.

Итоги
- Нет возможности случайного удаления — шаг удаления всегда подтверждается модалкой.
- Все изменения состояния ⭐ мгновенно синхронизированы во всех точках.
- Логика отображения и поведения не содержит противоречий: одно и то же действие всегда приводит к одному ожидаемому результату.
- Родной альбом является финальным арбитром присутствия трека в «Избранном».

Назначение кнопки F (Только избранные)
Кнопка F включает/выключает режим Favorites Only только для текущего playing-плейлиста: она никогда не ставит и не снимает ⭐, а лишь сужает доступный набор треков для Next/Prev/автоперехода/Shuffle и делает правило «что видно в playing-контексте — то и играет».
F всегда относится к playing-context, а не к просто открытому экрану; полный и мини-плеер управляют одним и тем же состоянием, поэтому поведение F везде идентично.
При F=ON в обычных альбомах/витрине доступны только ⭐-треки текущего playing-контекста, причём hidden в Showcase имеет приоритет выше ⭐, а если текущий трек не подходит — плеер мгновенно и без stop переходит на первый допустимый.
Если в текущем playing-контексте нет ни одного допустимого ⭐-трека, F не включается, показывается уведомление «отметьте ⭐ понравившийся трек», а текущее воспроизведение не ломается.
Если при F=ON пользователь запускает не⭐ трек из другого контекста, открывается модалка с вариантами: отключить F и воспроизвести, добавить ⭐ и продолжить в F, либо отменить; если запускается уже ⭐ трек, playing-context сразу перестраивается под новый список.
В `__favorites__` inactive-строки — это только UI-буфер для возврата/удаления и они никогда не участвуют в playback, Next/Prev, autoplay и Shuffle ни при F=OFF, ни при F=ON; там F фактически no-op по составу playback, но должен оставаться единым системным режимом.
Выключение F никогда не должно обрывать текущий трек: текущий uid сохраняется, а playing-плейлист просто расширяется обратно до полного source-context; повторное включение F, если текущий трек допустим, тоже не должно рвать звук.
Снятие последней ⭐ при F=ON вне favorites-view не имеет права останавливать плеер: F автоматически выключается, обычный playing-плейлист восстанавливается и музыка продолжает играть; stop разрешён только в одном сценарии — если в favorites-view текущий трек стал inactive и больше не осталось active.

name: Deploy App to Yandex Cloud S3

on:
  push:
    branches: [main, master]
    paths-ignore:
      - '.meta/**'
      - '.github/workflows/generate-context.yml'
  workflow_dispatch:

concurrency:
  group: yc-s3-deploy-${{ github.ref }}
  cancel-in-progress: true

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 2

      - name: Configure CORS
        if: github.event_name == 'workflow_dispatch'
        env:
          AWS_ACCESS_KEY_ID: ${{ secrets.YC_ACCESS_KEY_ID }}
          AWS_SECRET_ACCESS_KEY: ${{ secrets.YC_SECRET_ACCESS_KEY }}
          AWS_DEFAULT_REGION: 'ru-central1'
        run: |
          cat > /tmp/cors.json << 'EOF'
          {"CORSRules":[{"AllowedOrigins":["*"],"AllowedMethods":["GET","HEAD"],"AllowedHeaders":["*"],"MaxAgeSeconds":86400}]}
          EOF
          aws --endpoint-url=https://storage.yandexcloud.net s3api put-bucket-cors --bucket vi3na1bita --cors-configuration file:///tmp/cors.json

      - name: Incremental deploy public files
        env:
          AWS_ACCESS_KEY_ID: ${{ secrets.YC_ACCESS_KEY_ID }}
          AWS_SECRET_ACCESS_KEY: ${{ secrets.YC_SECRET_ACCESS_KEY }}
          AWS_DEFAULT_REGION: 'ru-central1'
          BEFORE_SHA: ${{ github.event.before }}
          BUCKET: vi3na1bita
        run: |
          set -euo pipefail

          is_public_file() {
            case "$1" in
              .git/*|.github/*|.meta/*|albums/*|scripts/ci/*|scripts/e2e/*|generate-context.js) return 1 ;;
              albums/gallery/*|audio/*|data/*|icons/*|img/*|news/*|scripts/*|src/*|styles/*) return 0 ;;
              index.html|oauth-callback.html|manifest.json|albums.json|service-worker.js) return 0 ;;
              *) return 1 ;;
            esac
          }

          upload_file() {
            local f="$1"
            [ -f "$f" ] || return 0
            is_public_file "$f" || return 0
            echo "PUT $f"
            aws --endpoint-url=https://storage.yandexcloud.net s3 cp "$f" "s3://$BUCKET/$f" --only-show-errors
          }

          delete_file() {
            local f="$1"
            is_public_file "$f" || return 0
            echo "DELETE $f"
            aws --endpoint-url=https://storage.yandexcloud.net s3 rm "s3://$BUCKET/$f" --only-show-errors || true
          }

          if [ "${{ github.event_name }}" = "workflow_dispatch" ] || [ -z "${BEFORE_SHA:-}" ] || [[ "$BEFORE_SHA" =~ ^0+$ ]]; then
            echo "Manual/initial deploy: full public sync"
            aws --endpoint-url=https://storage.yandexcloud.net s3 sync ./ "s3://$BUCKET/" --delete --only-show-errors \
              --exclude "*" \
              --include "index.html" \
              --include "oauth-callback.html" \
              --include "manifest.json" \
              --include "albums.json" \
              --include "service-worker.js" \
              --include "styles/*" \
              --include "scripts/*" \
              --include "scripts/**" \
              --include "src/*" \
              --include "src/**" \
              --include "data/*" \
              --include "data/**" \
              --include "icons/*" \
              --include "img/*" \
              --include "img/**" \
              --include "audio/*" \
              --include "news/*" \
              --include "albums/gallery/*" \
              --include "albums/gallery/**" \
              --exclude ".meta/*" \
              --exclude ".github/*" \
              --exclude "scripts/ci/*" \
              --exclude "scripts/e2e/*" \
              --exclude "generate-context.js"
            exit 0
          fi

          git diff --name-status "$BEFORE_SHA" "$GITHUB_SHA" | while IFS=$'\t' read -r status a b; do
            file="${b:-$a}"
            if [[ "$status" == D* ]]; then
              delete_file "$a"
            else
              upload_file "$file"
            fi
          done

      entries = fs.readdirSync(dir, { withFileTypes: true });
    } catch {
      return;
    }

    const visible = entries
      .filter(x => !isExcluded(toUnix(path.relative(ROOT, path.join(dir, x.name))), EXCLUDE_TREE_PATTERNS))
      .sort((a, b) => a.isDirectory() !== b.isDirectory() ? (a.isDirectory() ? -1 : 1) : a.name.localeCompare(b.name));

    visible.forEach((x, i) => {
      const last = i === visible.length - 1;
      lines.push(`${prefix}${last ? '└── ' : '├── '}${x.name}${x.isDirectory() ? '/' : ''}`);
      if (x.isDirectory()) walk(path.join(dir, x.name), `${prefix}${last ? '    ' : '│   '}`);
    });
  };

  walk(ROOT);
  return lines.join('\n');
};

const fileBlock = rel => `//=================================================
// FILE: /${toUnix(rel)}
${readText(rel)}
`;

const generate = mode => {
  let out = headerBlock();
  let lines = countLines(out);

  const groups = listAllEntries(true, false)
    .filter(e => !e.dir && isTextFile(e.rel))
    .reduce((acc, e) => {
      acc[getPriority(e.rel)].push(e.rel);
      return acc;
    }, { critical: [], high: [], medium: [], low: [] });

  const order = mode === 'adaptive' ? ['critical', 'high', 'medium'] : ['critical', 'high', 'medium', 'low'];

  for (const level of order) {
    for (const rel of groups[level]) {
      const block = fileBlock(rel);
      const blockLines = countLines(block);
      if (mode === 'adaptive' && lines + blockLines > MAX_LINES) {
        return `${out}\n// ... (truncate)\n`;
      }
      out += block;
      lines += blockLines;
    }
  }

  return out;
};

export const createGameBridgeHost = ({ iframe, config = {}, onState } = {}) => {
  const bridgeId = crypto.randomUUID();
  let alive = true;

  const send = (type, payload = {}) => {
    if (!alive || !iframe?.contentWindow) return false;
    try {
      iframe.contentWindow.postMessage({ kind: 'vitrina:game-host', bridgeId, type, payload }, '*');
      return true;
    } catch { return false; }
  };

  const sendSnapshot = () => send('GC_SNAPSHOT', buildSnapshot({ config }));

  const onMessage = e => {
    if (!alive || e.source !== iframe?.contentWindow) return;
    const d = e.data || {};
    if (d.kind !== 'vitrina:game' || d.bridgeId !== bridgeId) return;
    if (d.type === 'GC_READY' || d.type === 'GC_REQUEST_SNAPSHOT') sendSnapshot();

    if (d.type === 'GC_SAVE_DATA') {
      if (d.payload?.gameId && d.payload?.key) {
        const gcId = localStorage.getItem('intel:internal-user-id') || localStorage.getItem('deviceHash') || 'local';
        try {
          const root = JSON.parse(localStorage.getItem(`gc_data_${gcId}`) || '{}');
          root[`${d.payload.gameId}_${d.payload.key}`] = d.payload.data;
          localStorage.setItem(`gc_data_${gcId}`, JSON.stringify(root));
          sendSnapshot(); // Рассылаем всем клиентам обновление
        } catch {}
      }
      return;
    }

    if (d.type === 'GC_DOOR_CLICKED') {
      try {
        W.eventLogger?.log?.('FEATURE_USED', 'global', {
          feature: 'game_center_door',
          door: safe(d.payload?.door || ''),
          revision: safe(config.revision || '')
        });
      } catch {}
      return;
    }

    if (d.type === 'GC_PARENT_SCROLL') {
      const dy = Math.max(-160, Math.min(160, n(d.payload?.deltaY || 0)));
      if (dy) {
        try { W.scrollBy({ top: dy, left: 0, behavior: 'auto' }); } catch { W.scrollBy(0, dy); }
      }
      return;
    }

    if (d.type === 'GC_COLLAPSE_GAME') {
      const host = document.querySelector('.gc-host.is-mounted');
      if (host) {
        document.body.appendChild(host); // Спасаем iframe от уничтожения при рендере альбома
        host.dataset.gcCollapsed = '1';
        host.classList.add('is-gc-parked');
        host.style.display = '';
      }

      const pc = W.playerCore;
      const am = W.AlbumsManager;
      if (pc && am) {
        const track = pc.getCurrentTrack();
        if (track && track.sourceAlbum) {
          am.loadAlbum(track.sourceAlbum).then(() => {
            setTimeout(() => {
              const el = document.querySelector(`.track[data-uid="${CSS.escape(track.uid)}"]`);
              if (el) el.scrollIntoView({ behavior: 'smooth', block: 'center' });
            }, 300);
          });
        }
      }

      let floater = document.getElementById('gc-floating-heart');
      if (!floater) {
        floater = document.createElement('div');
        floater.id = 'gc-floating-heart';
        floater.innerHTML = `
          <div class="gc-floating-pulse"></div>
          <img src="img/icon_game.png" class="gc-floating-img" alt="Разбитое сердце">
          <div class="gc-floating-text">ВЕРНИСЬ<br>В ИГРУ</div>
          <button class="gc-floating-close" aria-label="Закрыть игру">✕</button>
        `;
        document.body.appendChild(floater);

        const restore = () => {
          floater.remove();

          const savedHost = host || document.querySelector('.gc-host.is-mounted');
          if (!savedHost) return;

          savedHost.dataset.gcCollapsed = '0';
          savedHost.classList.remove('is-gc-parked');
          savedHost.hidden = false;
          savedHost.style.display = '';
          savedHost.classList.add('is-mounted');

          const frameWrap = savedHost.querySelector('#gc-frame-wrap, .gc-frame-wrap');
          if (frameWrap) {
            frameWrap.hidden = false;
            frameWrap.style.display = '';
          }

          const panel = savedHost.querySelector('.gc-panel');
          if (panel) {
            panel.hidden = true;
            panel.style.display = 'none';
          }

          const frame = savedHost.querySelector('.gc-frame');
          const gameId = d.payload?.gameId || 'war_hearts';
          const post = (type, payload = {}) => {
            try {
              frame?.contentWindow?.postMessage({ kind: 'vitrina:game-host', bridgeId, type, payload }, '*');
            } catch {}
          };

          // Re-handshake: если Safari/iOS выгрузил JS-state Башни, она заново получит bridgeId и gameId.
          post('GC_INIT', { bridgeId, snapshot: buildSnapshot({ config }) });
          post('GC_RESTORE_GAME', { gameId, at: Date.now() });
          post('GC_SNAPSHOT', buildSnapshot({ config }));

          setTimeout(() => {
            post('GC_INIT', { bridgeId, snapshot: buildSnapshot({ config }) });
            post('GC_RESTORE_GAME', { gameId, at: Date.now() });
            post('GC_SNAPSHOT', buildSnapshot({ config }));
          }, 160);
        };

        floater.querySelector('.gc-floating-img').onclick = restore;
        floater.querySelector('.gc-floating-text').onclick = restore;

        floater.querySelector('.gc-floating-close').onclick = (e) => {
          e.stopPropagation();
          W.Modals?.confirm?.({
            title: 'Выход из игры',
            textHtml: 'Сессия прервётся, и не законченные игры не принесут очки. Точно выйти?',
            confirmText: 'Выйти',
            cancelText: 'Отмена',
            onConfirm: () => {
              floater.remove();

              const savedHost = host || document.querySelector('.gc-host.is-mounted');
              if (savedHost) savedHost.remove();

              onState?.({ state: 'closed_by_game' });
            }
          });
        };
      }
      return;
    }

    if (d.type === 'GC_CLOSE') {
      try {
        W.eventLogger?.log?.('FEATURE_USED', 'global', {
          feature: 'game_center_close',
          revision: safe(config.revision || '')
        });
      } catch {}
      onState?.({ state: 'closed_by_game' });
    }
  };

  const onHostUpdate = () => send('GC_HOST_STATE', buildSnapshot({ config }));

  W.addEventListener('message', onMessage);
  ['achievements:updated', 'stats:updated', 'analytics:liveTick', 'yandex:auth:changed', 'player:play', 'player:pause', 'player:stop', 'player:trackChanged'].forEach(x => W.addEventListener(x, onHostUpdate));

  iframe.addEventListener('load', () => {
    send('GC_INIT', { bridgeId, snapshot: buildSnapshot({ config }) });
    sendSnapshot();
  }, { once: true });

  return {
    bridgeId,
    sendSnapshot,
    destroy() {
      alive = false;
      W.removeEventListener('message', onMessage);
      ['achievements:updated', 'stats:updated', 'analytics:liveTick', 'yandex:auth:changed', 'player:play', 'player:pause', 'player:stop', 'player:trackChanged'].forEach(x => W.removeEventListener(x, onHostUpdate));
    }
  };
};

// FILE: /scripts/utils/sw-manager.js
(function (W, N, D) {
  'use strict';
  let refreshing = false;

  class SWManager {
    _clearUpdateUi() {
      D.getElementById('update-app-panel')?.remove();
      D.body.classList.remove('sw-update-visible');
    }

    async init() {
      if (!('serviceWorker' in N)) return;
      this._clearUpdateUi();

      // Не reload по controllerchange сам по себе. Reload только если пользователь явно нажал «ОБНОВИТЬ».
      N.serviceWorker.addEventListener('controllerchange', () => {
        this._clearUpdateUi();
        try { W.playerCore?._persistPlaybackState?.(true); } catch {}
        if (refreshing) {
          setTimeout(() => W.location.reload(), 80);
          return;
        }
        try { W.NotificationSystem?.info?.('Обновление применится при следующем запуске.', 3500); } catch {}
      });

      try {
        this.reg = await N.serviceWorker.register('./service-worker.js', { scope: './' });
        if (!this.reg.waiting) this._clearUpdateUi(); else this._notifyUpdate();
        this.reg.addEventListener('updatefound', () => {
          const nw = this.reg.installing;
          nw?.addEventListener('statechange', () => {
            if (nw.state === 'installed' && N.serviceWorker.controller) this._notifyUpdate();
          });
        });
      } catch (e) {
        console.warn('SW Register Failed:', e);
      }
    }

    handleVersionMessage(d) {
      if (d?.swVer && d.swVer !== (W.VERSION || '')) this._notifyUpdate();
    }

    _notifyUpdate() {
      if (D.getElementById('update-app-panel')) return;
      D.body.classList.add('sw-update-visible');
      W.NotificationSystem?.info?.('Доступно обновление приложения.', 3500);

      const panel = Object.assign(D.createElement('div'), {
        id: 'update-app-panel',
        className: 'sw-update-panel',
        innerHTML: `<button type="button" class="sw-update-btn sw-update-btn--apply">ОБНОВИТЬ</button><button type="button" class="sw-update-btn sw-update-btn--later">ПОЗЖЕ</button>`
      });

      panel.querySelector('.sw-update-btn--apply')?.addEventListener('click', () => this.applyUpdate());
      panel.querySelector('.sw-update-btn--later')?.addEventListener('click', () => {
        this._clearUpdateUi();
        W.NotificationSystem?.info?.('Обновится после перезапуска страницы.', 3500);
      });

      D.body.appendChild(panel);
    }

    applyUpdate() {
      refreshing = true;
      try { W.playerCore?._persistPlaybackState?.(true); } catch {}
      const waiting = this.reg?.waiting;
      if (waiting) {
        waiting.postMessage?.({ type: 'SKIP_WAITING' });
        setTimeout(() => { if (refreshing) W.location.reload(); }, 1800);
      } else {
        W.location.reload();
      }
    }

    getCacheSize() {
      if (!this.reg?.active) return Promise.resolve(0);
      return new Promise(r => {
        const ch = new MessageChannel();
        ch.port1.onmessage = e => r(e.data?.size || 0);
        setTimeout(() => r(0), 1000);
        this.reg.active.postMessage({ type: 'GET_CACHE_SIZE' }, [ch.port2]);
      });
    }
  }

  W.ServiceWorkerManager = new SWManager();
  D.readyState === 'loading' ? D.addEventListener('DOMContentLoaded', () => W.ServiceWorkerManager.init()) : W.ServiceWorkerManager.init();
})(window, navigator, document);
